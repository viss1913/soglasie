# Задача: Интеграция MAX Messenger и Конструктор команд

## 1. Личный кабинет агента (Подключение MAX)

Необходимо добавить возможность выбора типа мессенджера при создании/настройке бота.

### Настройка бота
- **Новое поле**: `bot_type` (Select: `telegram`, `max`). По умолчанию `telegram`.
- **Новое поле (только для max)**: `webhook_secret`. Текстовое поле (агент может задать сам для безопасности).
- **Индикация Webhook URL**: Если выбран тип `max`, показывать пользователю URL, который он должен прописать в кабинете MAX (если автоматическая регистрация не сработала или для проверки).
  - Формат: `https://[DOMAIN]/api/pfp/constructor/webhook/max/[BOT_ID]`

### Получение настроек
- `GET /api/pfp/constructor/bot` — теперь возвращает **массив** всех ботов агента.
- Фронтенд должен уметь:
  1. Выводить список ботов (например, вкладками "Telegram" и "MAX").
  2. Если ботов еще нет, предлагать создать.
  3. Если бот уже есть, выводить его настройки для редактирования.

### API для сохранения
```http
POST /api/pfp/constructor/bot
Content-Type: application/json

{
  "name": "Имя бота",
  "link": "...", 
  "token": "...",
  "bot_type": "max", // Обязательно передавать тип
  "webhook_secret": "...",
  "communication_style": "...",
  "base_brain_context": "..."
}
```
*Примечание: Если бот такого типа уже существует у агента, он будет обновлен. Если нет — создан новый.*

## 2. Админ-панель: Универсальный Конструктор команд (CJM)

Конструктор сценариев является **единым** для всех типов мессенджеров (Telegram, MAX). Логика классификации и ответов ИИ не зависит от платформы.

### Управление шаблонами (Глобальные команды)
- **Таблица**: Список всех глобальных команд (`is_template: true`).
- **Свойства**:
  - `command`: Ключ (например, `/start`, `/meeting`).
  - `section`: Группа (Продажи, Помощь).
  - `classifier`: Инструкция для ИИ (когда включать).
  - `response`: Инструкция для ИИ (что отвечать).

### Управление командами конкретного бота
- Позволяет переопределить или добавить команды для конкретного бота агента.
- API то же самое, просто передается `bot_id`.

## 3. Технические детали MAX API (Справка)

- **Типы событий**: Бэкенд обрабатывает `message_created`.
- **Авторизация**: Бот отправляет заголовок `Authorization: <token>`.
- **Вебхуки**: Бэкенд сам вызывает `POST /subscriptions` при сохранении настроек бота, чтобы зарегистрировать свой URL.
- **Подпись**: Бэкенд проверяет заголовок `X-Max-Bot-Api-Secret` на соответствие `webhook_secret` из БД.
