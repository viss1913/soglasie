# Интеграция Риск-Профилирования (Методика А. Деньгиной)

Для фронтенда реализована возможность сохранять ответы анкеты риск-профилирования. На основе этих ответов бэкенд **автоматически** рассчитывает риск-профиль (Conservative/Balanced/Aggressive) для каждой цели индивидуально, учитывая её срок.

## 1. Структура Анкеты

Анкета состоит из 10 вопросов.
- **Вопрос 1 (Инвестиционный Горизонт)**: Рассчитывается **автоматически на бэкенде** на основе срока цели (`term_months`). Фронтенду не нужно его присылать или спрашивать у пользователя в контексте конкретной цели.
- **Вопросы 2-10**: Ответы пользователя. Представляют собой числовые значения (баллы) от 1 до 5.

## 2. API Запросы

Ответы на вопросы 2-10 нужно передавать в объекте `client` внутри полей `risk_profile_answers`.

### POST `/api/my/plan/first-run` или `/api/my/plan/:id/recalculate`

```json
{
  "client": {
    "risk_profile_answers": {
      "q2": 3,
      "q3": 3,
      "q4": 4,
      "q5": 3,
      "q6": 3,
      "q7": 4,
      "q8": 3,
      "q9": 3,
      "q10": 4
    }
  },
  "goals": [
    {
      "goal_type_id": 4,
      "name": "Квартира",
      "target_amount": 10000000,
      "term_months": 240, 
      "initial_capital": 0
    }
  ]
}
```

> **Важно:** Поле `risk_profile` внутри объекта цели (`goals[i].risk_profile`) теперь является **необязательным** при отправке. Бэкенд сам заполнит его. Если вы передадите его, оно будет проигнорировано/перезаписано умным алгоритмом.

## 3. Логика расчёта (Для справки)

Бэкенд суммирует баллы:
`TotalScore = Points(Q1) + Sum(Q2...Q10)`

Баллы за Q1 (Срок цели):
- 1-2 года (<= 24 мес): **1 балл**
- 3-5 лет (<= 60 мес): **2 балла**
- 6-10 лет (<= 120 мес): **3 балла**
- 11-20 лет (<= 240 мес): **4 балла**
- > 20 лет: **5 баллов**

Итоговый профиль:
- 10-20 баллов: **CONSERVATIVE**
- 21-34 балла: **BALANCED**
- 35-50 баллов: **AGGRESSIVE**

## 4. Ответ Бэкенда

В ответе для каждой цели вернется рассчитанный профиль:

```json
{
  "goals": [
    {
      "goal_id": 15,
      "name": "Квартира",
      "risk_profile": "AGGRESSIVE", 
      ...
    }
  ]
}
```
